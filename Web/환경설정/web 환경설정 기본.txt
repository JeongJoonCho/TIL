웹 어플리케이션 개발을 목표로 함
-> MVC로 모든게 구성이 됨

기업형 프로그램을 만드려면 굉장히 많은 것들을 봐줘야한다.

그래서 스프링을 기반으로 만든다
스프링 ==> 프레임워크 (반자동화)

프레임워크 (반제품)
DI --> 컨테이너가 관장--> 자동화 되어 있다.

자동화: 개발자가 코드를 생성하지 않는것

*환경설정*

- xml : 	web.xml	- 컨트롤러위치와 웹 어플리케이션 기본 환경설정
	servlet-context.xml	- 요청처리와 아웃풋 처리하는 데 필요한 정보
	action-mybatis.xml	- 커넥션 풀 만들고, 쿼리자동화(저수준jdbc) 설정, VO정보위치 
		modelConfig.xml - VO정보위치
		mybatis.mappers - 쿼리자동화(저수준jdbc)
	db.properties - 데이터소스관리


*환경설정 후 해야할 일*
DAO에서 데이터베이스에 직접 연결이 되는가
컨트롤러가 실행이 되는가


*기본 스프링 환경에서 초기 에러 디버깅*

# pom.xml을 무조건 복사하지 않는다.
	- 기본으로 만들어진 것에 필요한 것을 한꺼번에 넣지 않고 하나씩 추가한다.
	- pom.xml이 에러가 났을 경우 반드시 에러를 잡은 후에 코딩한다.

# mysql의 경우 Timezone과 드라이버이름의 변경으로 인하여 주의한다. 
	-maven의 mysql 8.0.27 버전 드라이버를 사용한다.	

# 서비스, 컨트롤러, VO, DAO에 대한 어노테이션을 정확히 작성한다.

# 마이바티스의 modelConfig.xml에 적용되는 VO의 패키지명을 정확히 입력한다.(복사해서 쓸 경우 패키지명 틀림)
